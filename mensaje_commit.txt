fix: bugs editor quiz + mejoras UX cursos admin

## Bugs corregidos

### Editor de preguntas quiz
- Botón "Guardar" del quiz quedaba inactivo tras agregar preguntas
  Causa: estado local no se sincronizaba con DB tras operación exitosa
  Fix: actualización correcta del estado después de cada pregunta guardada

- IDs de alternativas se duplicaban al eliminar y reagregar opciones
  Causa: IDs (a,b,c,d) no se recalculaban tras eliminar una alternativa
  Fix: reasignación secuencial de IDs después de cada eliminación
  Fix: ID de nueva alternativa calculado desde options.length actual
  Fix: correct_option se resetea si se elimina la alternativa correcta

- Error de consola "Encountered two children with the same key 'd'"
  Causa: duplicación de IDs en el map de alternativas
  Fix: garantizado por recálculo secuencial de IDs

- Botón eliminar alternativa deshabilitado cuando quedan solo 2 opciones
  Previene estructura rota e inconsistencias en el formulario

## Mejoras UX

### Vista cursos admin
- Botón "Ver Curso" renombrado a "Ver reporte"
- Redirige a /admin/reportes?curso=[id] en vez del dashboard
- Al llegar desde este botón los reportes se filtran automáticamente
  por el título del curso seleccionado

### Reportes
- ReportesClient lee parámetro ?curso= de la URL al montar
- Inicializa el filtro de búsqueda con el título del curso
  cuando viene desde la vista de cursos admin
- Prop courses añadida a ReportesClient para resolver el título
